+ 브랜치를 나눠서 작업후 병합을 하거나
+ 원격 저장소를 이용해 협업을 하거나
+ 형상관리를 해야 하는데, 기준점이 두개일때는 어떻게 하나요?

```
git switch 기준이 될 브랜치 이름
git merge 병합할 브랜치 이름

# 충돌이 일어났다면, 해당 파일을 해결
git add .
git commit -m "충돌 해결 내용 작성"

# 병합 완료된 브랜치 삭제하기
git branch -d 삭제할 브랜치 이름
```

+ 충돌을 발생시키고, 병합하기
    + 브랜치 생성하기
        ```
        git branch 새로운 브랜치 이름
        ```
    + master와 새 브랜치에서 같은 파일의 같은 라인을 각각 다르게 편집
    + master를 기준으로 새 브랜치를 병합
        ```
        git switch master
        git merge 병합할브랜치이름
        # -> 충돌이 일어나면 더이상 진행이 안됩니다
        ```
    + 충돌이 일어난 파일이 '기본 편집기'에서 열려있을겁니다
        ```
        <<<<<<< HEAD 
        현재 내 브랜치(master)의 변경내용
        =======
        가져오려는 브랜치의 변경내용
        >>>>>>>
        ```
    + 두 내용 중 남길 코드를 결정
    + '<<<<<<<', '=======','>>>>>>>' 표시를 포함하여, 필요없는 내용을 삭제
    + 저장
    + add -> commit
    + 옵션 : 병합 완료된 브랜치 삭제
        ```
        git branch -d 삭제할브랜치이름
        ```
